## 1) 信息架构重组（New Tab 最适合的结构）

### 推荐：三段式布局（最像 macOS 的工作流）

- **Top Toolbar（固定）**：全局搜索 + 视图切换 + 导入导出/刷新/设置
    
- **Left Sidebar（可收起）**：文件夹 / 标签 / 智能集合（Tab 切换）
    
- **Main Canvas（主内容）**：书签网格/列表（置顶 + 最近 + 当前文件夹/搜索结果）
    
- **Right Inspector（默认隐藏）**：点击某条书签后滑出，展示详情与编辑操作（替代你现在右侧常驻大面板）
    

> 你的“常用书签”“统计数据”不要常驻占空间：
> 
> - **常用书签 → 智能集合（Smart Collection）**（在侧边栏里）
>     
> - **统计数据 → 智能分析中心**里展示（你顶部已经有入口了）


## 2) Top Toolbar（苹果感的核心）

Apple 的工具栏强调：把高频命令放在顶部工具栏，避免到处都是面板按钮。

**布局（高度建议 56px）**

- 左：Logo + 当前区域（概览 / 文件夹管理 / 智能分析中心）
    
- 中：**全局搜索框**（默认焦点支持 Cmd+K，但鼠标为主也要点一下就能用）
    
- 右：
    
    - 视图切换 Segmented：`卡片` / `列表`
        
    - Icon Buttons：`导入导出` `刷新` `设置`
        

**搜索框行为（强烈建议做“Spotlight感”）**

- 输入时下拉结果分组：`书签` / `文件夹` / `标签`
    
- 顶部提供过滤 chips：`在当前文件夹` `仅置顶` `标签:` `域名:`
    
- 支持清除按钮与占位提示（Apple 搜索字段典型元素：放大镜、清除）


## 3) Sidebar（鼠标友好的导航）

**宽度 260–280px；支持一键收起**（只留图标栏也行）

顶部做 Tabs：

- **文件夹**：树状结构 + 计数（计数弱化成 secondary 文本）
    
- **标签**：支持“常用标签置顶”
    
- **智能**：常用、最近7天、未分类、重复域名、无favicon、稍后读（你现有能力都能组合出来）
    

交互细节（鼠标为主）：

- Hover：显示“更多(···)”
    
- 右键菜单：重命名/新建子文件夹/导出该文件夹/删除
    
- 拖拽到文件夹：hover 500ms 自动展开（整理会爽很多）
    

侧边栏属于典型侧栏控件形态，可参考 Apple 的侧边栏设计要点（层级清晰、弱装饰、靠选中态强调）。

## 4) 主内容区（Main Canvas）：让“丑”立刻变“干净”的关键

### 内容结构（从上到下）

1. **Pinned（置顶，可折叠）**
    
2. **Recent（最近）** 或 **当前文件夹内容** 或 **搜索结果**
    
3. 分页/懒加载（新标签页性能要稳）
    

### 两种视图（你已经有图标切换雏形）

#### A) 卡片视图（默认）

- 卡片高度：**72px**（统一高度很“高级”）
    
- 左：favicon 24px
    
- 中：标题 1 行（Semibold 14–15）+ 域名 1 行（12–13 secondary）
    
- 右：hover 才出现：`置顶图钉` + `···`
    
- 点击卡片：**打开链接**
    
- 点击 `···`：弹出菜单（编辑/复制链接/移动/加标签/删除）
    

> 你现在卡片黄底+重描边会显“廉价”：建议改成白底卡片 + 极轻分隔（阴影和边框二选一，别两者都上）。

#### B) 列表视图（效率向）

- 列：标题 | 域名 | 标签 | 最后访问
    
- 表头可排序：最近/常用/字母

## 5) 右侧 Inspector（替代现在右边“常用书签”大面板）

默认隐藏，**点中某条书签后右侧滑出 360px**：

内容建议：

- 标题（可编辑）
    
- URL（复制按钮）
    
- 所属文件夹（下拉选择/移动）
    
- 标签（chips，可增删）
    
- 备注（可选）
    
- 操作：置顶 / 新窗口打开 / 删除
    
- 元信息：创建时间、最后访问、访问次数（如果你有）
    

这样右侧不再“常驻抢视线”，但功能更强。

## 6) 鼠标为主的关键交互（做出“苹果感”）

### 右键上下文菜单（强烈建议全覆盖）

- 空白处右键：新建文件夹/导入/刷新/视图切换
    
- 书签右键：打开/新窗口打开/复制链接/编辑/移动/加标签/置顶/删除
    
- 文件夹右键：重命名/导出/新建子文件夹/删除
    

### 拖拽手感（提升整理效率）

- 拖书签到文件夹：文件夹高亮 + hover 展开
    
- 拖书签到标签：添加标签（不是移动）
    
- 拖动排序：只在“置顶区”或“常用集合”里允许排序，避免误操作
    

### Hover 反馈（克制但明确）

- 卡片 hover：背景轻微变亮 + 右侧操作淡入
    
- 置顶：图钉 hover 出现；置顶后保持可见（颜色更深/品牌色）

## 7) 视觉 token（你可以直接丢进 CSS/设计系统）

- Grid/Spacing：8 / 16 / 24 / 32
    
- 圆角：Card 12，Button/Input 10
    
- 字号：12（辅助）/ 14（正文）/ 16-18（区块标题）
    
- 分隔线：1px 超浅（比现在淡很多）
    
- 背景：非常浅灰；卡片：白
    
- 强调色：保留你的橙色，但只用于：主按钮、选中态、小面积点缀
    

---

## 8) 浏览器扩展新标签页注意点（不改功能但要规避坑）

- Chrome 的新标签页覆盖属于扩展的“override”能力（你应该已经在用），并且**无痕窗口的新标签页不能被覆盖**（需要在体验上给个提示或降级方案）。
